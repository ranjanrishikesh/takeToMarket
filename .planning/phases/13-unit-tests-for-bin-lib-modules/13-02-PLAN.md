```javascript
describe('state.cjs', () => {
  let tmp, originalCwd, stdoutMock, stderrMock, exitMock;

  before(() => {
    tmp = createTempDir();
    createMockMarketing(tmp.dir);  // Creates .marketing/STATE.md with status: active
    originalCwd = process.cwd;
    process.cwd = () => tmp.dir;
  });

  after(() => {
    process.cwd = originalCwd;
    tmp.cleanup();
  });

  beforeEach(() => {
    stdoutMock = mock.method(process.stdout, 'write', () => true);
    stderrMock = mock.method(process.stderr, 'write', () => true);
    exitMock = mock.method(process, 'exit', () => {});
  });

  afterEach(() => {
    stdoutMock.mock.restore();
    stderrMock.mock.restore();
    exitMock.mock.restore();
  });

  // ... nested describes
});
```

1. **`describe('module exports')`**:
   - `it('exports cmdStateRead and cmdStateUpdate as functions')` -- assert both are functions

2. **`describe('cmdStateRead')`**:
   - `it('reads STATE.md and outputs frontmatter as JSON')` -- call `cmdStateRead(false)`, parse stdout JSON, assert `parsed.exists === true` and `parsed.status === 'active'`
   - `it('includes body_preview in output')` -- call `cmdStateRead(false)`, parse JSON, assert `parsed.body_preview` is a string containing `'Marketing State'`
   - `it('outputs raw JSON string when raw=true')` -- call `cmdStateRead(true)`, assert stdout contains a string (raw mode outputs `JSON.stringify(frontmatter)`)

Top-level describe with real git repo setup:

```javascript
describe('commit.cjs', () => {
  let tmp, originalCwd, stdoutMock, stderrMock, exitMock;

  before(() => {
    tmp = createTempDir();
    originalCwd = process.cwd;
    process.cwd = () => tmp.dir;
    // Init a real git repo in temp dir
    execSync('git init', { cwd: tmp.dir, stdio: 'pipe' });
    execSync('git config user.email "test@test.com"', { cwd: tmp.dir, stdio: 'pipe' });
    execSync('git config user.name "Test"', { cwd: tmp.dir, stdio: 'pipe' });
  });

  after(() => {
    process.cwd = originalCwd;
    tmp.cleanup();
  });

  beforeEach(() => {
    stdoutMock = mock.method(process.stdout, 'write', () => true);
    stderrMock = mock.method(process.stderr, 'write', () => true);
    exitMock = mock.method(process, 'exit', () => {});
  });

  afterEach(() => {
    stdoutMock.mock.restore();
    stderrMock.mock.restore();
    exitMock.mock.restore();
  });

  // ... nested describes
});
```

1. **`describe('module exports')`**:
   - `it('exports cmdCommit as a function')` -- assert `typeof cmdCommit === 'function'`

2. **`describe('cmdCommit validation')`** -- Tests that trigger errors BEFORE git operations:
   - `it('errors on empty message')` -- call `cmdCommit('', ['file.txt'], false)`, assert `exitMock.mock.calls.length >= 1`
   - `it('errors on whitespace-only message')` -- call `cmdCommit('   ', ['file.txt'], false)`, assert exit called
   - `it('errors on empty files array')` -- call `cmdCommit('msg', [], false)`, assert exit called
   - `it('errors on null files')` -- call `cmdCommit('msg', null, false)`, assert exit called
   - `it('errors on message exceeding 500 characters')` -- call `cmdCommit('a'.repeat(501), ['file.txt'], false)`, assert exit called
   - `it('errors on path traversal in file list')` -- call `cmdCommit('msg', ['../../etc/passwd'], false)`, assert exit called with exit code 1

3. **`describe('cmdCommit with real git repo')`** -- Tests that actually commit:
   - `it('commits a file and outputs JSON with sha')` -- create file `path.join(tmp.dir, 'test1.txt')` with content `'hello'`, call `cmdCommit('test commit', ['test1.txt'], false)`, parse stdout JSON, assert `parsed.committed === true` and `parsed.message === 'test commit'` and `parsed.sha` is a string of length >= 6. Verify with `execSync('git log --oneline', { cwd: tmp.dir })` that the commit exists.
   - `it('sanitizes shell metacharacters from commit message')` -- create file `path.join(tmp.dir, 'test2.txt')` with content `'data'`, call `cmdCommit('hello `whoami` $(rm -rf /) ${HOME}; echo; | cat', ['test2.txt'], false)`, parse stdout JSON, assert `parsed.message` does NOT contain backtick, `$(`, `${`, semicolon, or pipe characters. Assert `parsed.committed === true`.
   - `it('outputs raw sha when raw=true')` -- create file `path.join(tmp.dir, 'test3.txt')` with content `'raw'`, call `cmdCommit('raw test', ['test3.txt'], true)`, assert stdout contains a short hash string (not JSON).
   - `it('errors on message that becomes empty after sanitization')` -- call `cmdCommit('`; |', ['test1.txt'], false)`. The sanitized message becomes empty (all special chars replaced with spaces, then trimmed). Assert exit called.

**Critical notes:**
- Each test that successfully commits needs a NEW file (or modified file) because git won't commit if there are no staged changes. Use unique filenames: test1.txt, test2.txt, test3.txt, etc.
- `cmdCommit` uses `execFileSync('git', ...)` which executes git in the CURRENT process.cwd(). Since we override `process.cwd` to point to tmp.dir, and we init git there, the git commands will operate in the temp repo.
- IMPORTANT: `execFileSync` in commit.cjs does NOT pass `{ cwd: ... }` -- it relies on the process cwd. The cwd override MUST be set before calling cmdCommit.
- sanitizeMessage is NOT exported. Test it indirectly by verifying cmdCommit's output message field.
- The path traversal test: `../../etc/passwd` resolves outside `projectRoot` (which is `tmp.dir`), so cmdCommit calls error().
